# 资产页网格缩略图 1:1 化设计

## 1. 背景与目标
当前资产页网格缩略图容器采用 `aspect-video`（16:9）。需求是仅在资产页网格模式下将缩略图改为 1:1，并保持列表模式不变。

目标：
- 资产页网格缩略图统一显示为 1:1 正方形。
- 继续使用 `object-cover` 做裁切填充。
- 不改后端、不改数据结构、不改列表模式。

## 2. 范围确认（已冻结）
- 仅影响 `activeMenu === 'assets'` 下 `viewMode === 'grid'` 的卡片容器比例。
- 批量、收藏、筛选、删除、下载等交互保持不变。
- 详情弹层和列表模式不变。

## 3. 方案对比
### 方案 A（采用）
直接把资产网格卡片类名从 `aspect-video` 改为 `aspect-square`。

优点：
- 改动最小。
- 风险最低。
- 立即生效。

缺点：
- 比例写在 JSX，后续可配置性一般。

### 方案 B（未采用）
抽常量管理比例类名。

### 方案 C（未采用）
封装视图模式到类名映射函数。

## 4. 详细设计
修改文件：`src/App.tsx`

修改位置：资产页网格列表 `sortedAssets.map` 外层卡片容器 class 三元表达式。

- 现状：`viewMode === 'grid' ? 'aspect-video' : 'flex h-16 ...'`
- 目标：`viewMode === 'grid' ? 'aspect-square' : 'flex h-16 ...'`

保持不变：
- 图片 `img` 使用 `object-cover`。
- 列表模式尺寸 `w-20 h-12`。
- 选择态、hover、批量操作栏等行为。

## 5. 验证策略
- `npm run lint`
- `npm run build`
- 手工验证：
  - 资产页网格模式缩略图为 1:1。
  - 批量/收藏/筛选状态下仍为 1:1。
  - 列表模式视觉不变。

## 6. 非目标
- 不修改详情页大图比例。
- 不引入新的布局模式或样式变量系统。
- 不修改 API 或数据模型。
