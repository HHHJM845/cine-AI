# 生成预览瀑布流与比例选项精简设计

## 背景
当前生成界面右侧预览区域使用固定 `aspect-video` 容器与 `object-cover`，导致图片被裁切，无法按原图比例完整预览。\
同时比例下拉包含 `2.39:1`、`3:4`、`4:3` 等选项，不符合当前产品需求。

## 目标
1. 右侧预览区域中的生成结果改为按原图比例等比缩小展示，不裁切。\
2. 比例选项仅保留 `16:9` 与 `9:16`。

## 非目标
1. 不改动详情页大图展示逻辑。\
2. 不引入新依赖（如第三方瀑布流库）。\
3. 不调整生成接口协议字段。

## 方案对比

### 方案 A（推荐）：CSS 列布局瀑布流 + 等比缩小
- 做法：将批次图片区域由固定 4 列 `grid` 改为 `columns-*` 瀑布流；图片卡片使用 `break-inside-avoid` 防止断列；图片从 `object-cover` 改为 `object-contain` + `h-auto`。
- 优点：改动小、无依赖、可快速满足“按原图比例缩小预览”的核心诉求。
- 缺点：不同图片高度不一致，视觉节奏更自由但不规则。

### 方案 B：保留网格，动态 `aspect-ratio`
- 做法：读取图片尺寸后给每项设置 `aspect-ratio`。
- 优点：比例准确。
- 缺点：实现复杂度更高，对现有布局冲击更大。

### 方案 C：引入第三方 Masonry 组件
- 做法：集成专用瀑布流库。
- 优点：特性完整。
- 缺点：增加依赖与维护成本，超出当前必要范围。

## 选型
采用方案 A。

## 详细设计

### 1) 右侧预览区域（生成批次图片）
- 位置：`src/App.tsx`，`/* Batch Images */` 区块。
- 改造点：
  1. 容器 `grid grid-cols-4 gap-1` 改为响应式瀑布流列布局（如 `columns-1 sm:columns-2 lg:columns-3 xl:columns-4`）。
  2. 每个图片项增加 `break-inside-avoid` 与纵向间距。
  3. 成功图卡片移除固定 `aspect-video`，改为 `w-full`。
  4. 图片改为 `w-full h-auto object-contain`，确保完整显示原图比例。
  5. 失败卡片也使用同一列流项结构，避免布局异常。

### 2) 比例选项精简
- 位置：`src/App.tsx`
- 改造点：
  1. `SUPPORTED_ASPECT_RATIOS` 改为仅 `['16:9', '9:16']`。
  2. `aspectRatio` 默认值改为 `16:9`。
  3. 比例下拉渲染列表同步只显示上述两项。

## 影响评估
- 正向：预览图不再裁切，用户可直接对比原图构图比例；比例选择更聚焦。
- 风险：瀑布流下 hover 按钮与点击区域需复核，防止因容器结构变化导致错位。

## 验证策略
1. `npm run lint` 通过。\
2. `npm run build` 通过。\
3. 手动验证：
   - 生成不同宽高比图片后，右侧预览均完整显示且不裁切。
   - 瀑布流在常见窗口宽度下列数与排版正常。
   - 比例下拉只剩 `16:9` 和 `9:16`，默认为 `16:9`。
